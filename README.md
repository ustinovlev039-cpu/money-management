# Personal Finance Analyzer

Проект для анализа банковских транзакций из Excel-файла. Приложение формирует JSON-ответы для условных веб-страниц, рассчитывает расходы и поступления, анализирует категории кешбэка, моделирует работу «Инвесткопилки», выполняет поиск по операциям и создаёт отчёты.

## Возможности

### Главная страница

Функция `home_page()` формирует JSON со следующими данными:

* приветствие в зависимости от времени;
* информация по банковским картам:

  * последние четыре цифры карты;
  * общая сумма расходов;
  * кешбэк из расчёта 1 рубль на каждые 100 рублей трат;
* топ-5 транзакций по сумме платежа;
* курсы валют из пользовательских настроек;
* цены акций из пользовательских настроек.

### Страница событий

Функция `events_page()` формирует JSON с аналитикой за выбранный период:

* расходы;
* поступления;
* основные категории расходов;
* топ-7 категорий расходов;
* объединение остальных категорий в категорию `Остальное`;
* отдельные блоки для категорий `Переводы` и `Наличные`;
* курсы валют;
* цены акций.

Поддерживаемые периоды:

* `W` — текущая неделя;
* `M` — текущий месяц;
* `Y` — текущий год;
* `ALL` — все данные до указанной даты.

### Сервисы

В модуле `src/services.py` реализованы:

* анализ выгодных категорий повышенного кешбэка;
* расчёт суммы для «Инвесткопилки»;
* простой поиск по описанию и категории;
* поиск транзакций с мобильными номерами;
* поиск переводов физическим лицам.

### Отчёты

В модуле `src/reports.py` реализованы отчёты:

* траты по выбранной категории за последние три месяца;
* средние траты по дням недели;
* средние траты в рабочие и выходные дни.

Результаты отчётов сохраняются в JSON-файлы с помощью декоратора.

## Настройка пользовательских данных

Создай файл `data/user_settings.json` со списком валют и акций, которые должны отображаться в JSON-ответах.

Пример:

```json
{
    "user_currencies": ["USD", "EUR"],
    "user_stocks": ["AAPL", "AMZN", "GOOGL", "MSFT", "TSLA"]
}
```

Поля:

* `user_currencies` — список кодов валют;
* `user_stocks` — список тикеров акций.

---

## Настройка API-ключа для акций

Для получения стоимости акций используется Alpha Vantage API.

Создай в корне проекта файл `.env`:

```env
ALPHA_VANTAGE_API_KEY=your_api_key_here
```

Файл `.env` не должен добавляться в Git, поэтому он указан в `.gitignore`.

Если API-ключ не задан или внешний сервис временно недоступен, приложение не завершится с ошибкой: проблема будет записана в лог, а список акций может вернуться пустым.

## Формат Excel-файла

Приложение ожидает файл:

```text
data/operations.xlsx
```

В Excel должны присутствовать следующие столбцы:

| Столбец          | Назначение                                          |
| ---------------- | --------------------------------------------------- |
| `Дата операции`  | Дата и время выполнения операции                    |
| `Номер карты`    | Номер карты для отображения последних четырёх цифр  |
| `Статус`         | Статус операции, например `OK`                      |
| `Сумма операции` | Сумма операции для сервисов и расчёта Инвесткопилки |
| `Сумма платежа`  | Сумма платежа для страниц и отчётов                 |
| `Категория`      | Категория операции                                  |
| `Описание`       | Текстовое описание операции                         |

Правила обработки данных:

* успешными считаются операции со статусом `OK`;
* отрицательная сумма означает расход;
* положительная сумма означает поступление;
* для расходов в итоговых отчётах обычно используется положительное значение суммы;
* последние четыре цифры карты берутся из поля `Номер карты`.

## Запуск проекта

Запуск с параметрами:

```bash
python -m src.main \
  --date "2021-12-31 23:59:59" \
  --period M \
  --category "Супермаркеты" \
  --search "Ozon" \
  --investment-limit 50
```

Если параметр `--date` не передан, программа использует последнюю дату из файла `operations.xlsx`.

Это важно для учебного файла с историческими операциями: при запуске с текущей датой данные могут не попасть в анализируемый период.

---

## Аргументы командной строки

| Аргумент             | Назначение                       | Допустимые значения                                                      | По умолчанию            |
| -------------------- | -------------------------------- | ------------------------------------------------------------------------ | ----------------------- |
| `--date`             | Конечная дата анализа            | `YYYY-MM-DD HH:MM:SS`, `YYYY-MM-DD`, `DD.MM.YYYY HH:MM:SS`, `DD.MM.YYYY` | Последняя дата из Excel |
| `--period`           | Период для страницы событий      | `W`, `M`, `Y`, `ALL`                                                     | `M`                     |
| `--category`         | Категория для отчёта             | Любая строка                                                             | `Супермаркеты`          |
| `--search`           | Текст для простого поиска        | Любая строка                                                             | `Ozon`                  |
| `--investment-limit` | Шаг округления для Инвесткопилки | `10`, `50`, `100`                                                        | `50`                    |
| `--output-dir`       | Папка для JSON-результатов       | Путь к папке                                                             | `output/`               |

После выполнения команды в папке `output/` будут созданы файлы:


## Ограничения

* Данные для анализа берутся из локального Excel-файла.
* Курсы валют и цены акций зависят от доступности внешних API.
* Для получения цен акций необходим корректный API-ключ Alpha Vantage.
* При отсутствии данных за выбранную дату JSON-ответы могут содержать пустые списки.
* При работе с историческим файлом `operations.xlsx` рекомендуется указывать дату из диапазона имеющихся операций.
